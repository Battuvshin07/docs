13-р зууны Монголын түүхийг таниулах апп-ын гадаад дизайн нь хэрэглэгчдэд хялбар, интерактив туршлага өгөхөд чиглэсэн бөгөөд боловсролын зорилготой тул оффлайн дэмжлэгтэй, аюулгүй байх ёстой.

* **Гол онцлогууд**: Апп нь түүхэн хүмүүсийн намтар, байлдан дагуулалтын газрын зураг, цагийн хэлхээс,асуулт зэргийг багтаасан.  
* **Технологийн сонголт**: Flutter front-end ашиглан кросс-платформ (Android, iOS) хөгжүүлэлт, SQLite local DB-тэй, Firebase-ээр онлайн sync хийх.  
* **Хэрэглэгчийн туршлага**: Интерфейс нь Монгол хэл дээр, зөн совинтой дизайнтай, залуучуудад тохирсон.  
* **Аюулгүй байдал**: Local data encryption ашиглаж, хэрэглэгчийн мэдээллийг хамгаалах.

**Дизайны үндэслэл**

Апп нь боловсролын апп-ын стандарт шаардлагад нийцүүлэн хийгдсэн бөгөөд оффлайн горимд ажиллах чадвартай, хэрэглэхэд хялбар байхыг зорьсон. Жишээ нь, UI дизайн нь залуу хэрэглэгчдийн анхаарлыг татах зураг, интерактив элементүүдийг агуулна.

**Техникийн тохиргоо**  
Front-end: Flutter framework ашиглан UI/UX дизайн хийх.  
Back-end: Firebase-ээр өгөгдөл sync хийх (оффлайн дэмжлэгтэй).

Өгөгдлийн сан: SQLite local storage, Firestore cloud-д нөөцлөх.

**Аюулгүй байдлын товч**

Хэрэглэгчийн өгөгдлийг шифрлэх, API key-г аюулгүй хадгалах, input validation хийх зэрэг арга хэмжээ авна. Энэ нь оффлайн апп-д тохирсон, өгөгдлийн алдагдлаас сэргийлнэ.

### **1\. Системийн тойм**

Энэ хэсэгт системийн ерөнхий танилцуулга, зорилго, функцуудыг тодорхойлно. Дизайн нь боловсролын апп-ын стандарт SRS загварыг дагаж, хэрэглэгчийн туршлагыг сайжруулахад чиглэсэн.

#### **Системийн нэр**

13-р зууны Монголын түүхийг таниулах апп (Англиар: Mongol History Explorer App).

#### **Хэрэглэгчийн нэр**

* Үндсэн хэрэглэгч: Залуучууд, сурагчид, оюутнууд (13+ насны).  
* Нэмэлт: Багш нар, түүх судлаачид. Хэрэглэгчид нь Монгол хэл мэдэх, мобайл төхөөрөмжтэй байх шаардлагатай.

#### **Системийн зорилго**

Апп нь 13-р зууны Монголын эзэнт гүрний түүхийг (Чингис хаан, байлдан дагуулалт, соёл) интерактив байдлаар таниулах замаар залуу үеийнхэнд түүхийн мэдлэгийг түгээх, сонирхлыг нэмэгдүүлэхэд чиглэнэ. Оффлайн дэмжлэгтэй тул интернетгүй орчинд ашиглах боломжтой.

#### **Системийн функцууд**

* Түүхэн хүмүүсийн намтар, зураг харуулах.  
* Байлдан дагуулалтын интерактив газрын зураг.  
* Үйл явдлуудын цагийн хэлхээс.  
* Мэдлэг шалгах квиз, оноо цуглуулах.  
* Мультимедиа (зураг, видео, аудио) үзэх.  
* Хаадын гэр бүлийн мод харуулах.  
* Соёл, нийгмийн амьдралын мэдээлэл таниулах. Функцууд нь оффлайн горимд ажиллах бөгөөд онлайн болоход шинэчлэлт хийх боломжтой.

### **2\. Системийн тохиргоо**

Систем нь кросс-платформ байх тул Android болон iOS-д тохирсон. Оффлайн архитектур дээр суурилсан.

#### **Front-End**

Flutter framework ашиглан UI/UX дизайн хийх. Интерфейс нь Material Design эсвэл Cupertino загварыг дагаж, Монгол хэл дээр ажиллана. Оффлайн кэшинг ашиглан хурдан ажиллах.

#### **Арын төгсгөл**

Firebase эсвэл AWS AppSync ашиглан өгөгдөл sync хийх. Голчлон оффлайн тул local logic дээр суурилсан, онлайн болоход шинэ контент татах.

#### **Өгөгдлийн сан**

Local: SQLite. Cloud: Firestore. Өгөгдлийг локалд хадгалж, sync хийх.

### **3\. Дэлгэцийн дизайн**

Дэлгэцүүд нь интерактив, зөн совинтой байх ёстой. Wireframe болон mockup ашиглан дизайн хийх (Figma эсвэл Adobe XD). Дэлгэцүүдийн жишээ:

#### **Гол дэлгэц**

* Меню: Хүмүүс, Үйл явдал, Газрын зураг, Квиз, Соёл.  
* Дизайн: Кард загвартай, зурагтай жагсаалт.

#### **Түүхэн хүмүүсийн дэлгэц**

* Жагсаалт: Чингис хаан гэх мэт.  
* Дэлгэрэнгүй: Намтар, зураг, холбоосууд.

#### **Байлдан дагуулалтын газрын зургийн дэлгэц**

* Интерактив зураг: Зум хийх, тулаан дээр дарах.

#### **Интерактив цагийн хэлхээсийн дэлгэц**

* Timeline widget: Үйл явдлуудыг цаг хугацаагаар харуулах.

#### **Квиз дэлгэц**

* Асуулт хариулт, оноо харуулах, дахин тоглох.

#### **Соёлын мэдээллийн дэлгэц**

* Текст, зураг, видео багтаасан.

### **4\. Өгөгдлийн сангийн дизайн**

Өгөгдлийн сан нь relational байх бөгөөд SQLite-д тохирсон. Entity relationships: One-to-many хүмүүсийн үйл явдал.

| Хүснэгт | Талбарууд | Тайлбар |
| ----- | ----- | ----- |
| Persons | person\_id (PK), name, birth\_date, death\_date, description, image\_url | Түүхэн хүмүүсийн мэдээлэл. |
| Events | event\_id (PK), title, date, description, person\_id (FK) | Үйл явдлууд, холбоотой хүмүүстэй. |
| Maps | map\_id (PK), title, coordinates, event\_id (FK) | Газрын зургийн өгөгдөл. |
| Quizzes | quiz\_id (PK), question, answers, correct\_answer | Квиз асуултууд. |
| Media | media\_id (PK), type (image/video), url, related\_id (FK) | Мультимедиа файлууд. |

### **5\. Сүлжээний дизайн**

Оффлайн голчлон тул локал storage ашиглах. Онлайн болоход Firebase-ээр sync хийх: Periodic sync, conflict resolution. Сүлжээ: HTTPS протокол, API calls зөвхөн шинэчлэлтэд.

### **6\. Хамгаалалтын дизайн**

Оффлайн апп-д зориулсан:

* Өгөгдлийг локалд шифрлэх (AES).  
* Authentication: Credential Manager ашиглах.  
* Input validation: SQL injection-ээс сэргийлэх.  
* Permissions: Зөвхөн шаардлагатай зөвшөөрөл авах.  
* API key: Keystore-д хадгалах. Энэ нь өгөгдлийн алдагдлаас сэргийлж, хэрэглэгчийн нууцлалыг хангана.
